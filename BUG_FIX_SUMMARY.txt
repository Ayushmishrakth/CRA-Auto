================================================================================
                          THE PROBLEM
================================================================================

SYMPTOM:
  "Network Error x15" when clicking "Generate & Download" on customized reports

ROOT CAUSE:
  Manual Content-Type header in reportApi.js broke Axios's multipart encoding

THE BUG:
  Location: CRA-frontend/src/api/reportApi.js, line 25-27

  headers: {
    "Content-Type": "multipart/form-data",  // ❌ This breaks it!
  }

WHY IT FAILS:
  1. Axios needs to add a "boundary" marker when sending FormData
  2. The boundary separates different form fields in the request
  3. Manual header prevents Axios from adding the boundary
  4. Backend receives malformed multipart data
  5. FastAPI returns 422 (Unprocessable Entity)
  6. Frontend retries 15 times
  7. User sees "Network Error x15" and frustrated

================================================================================
                          THE SOLUTION
================================================================================

REMOVE THE HEADER:
  ❌ WRONG:
    headers: {
      "Content-Type": "multipart/form-data",
    }

  ✅ CORRECT:
    // Remove the headers object entirely
    // Let Axios detect FormData and set Content-Type automatically
    // It will automatically set: "multipart/form-data; boundary=..."

ADD ERROR LOGGING:
  ✅ Wrap in try/catch to show real error in console
  ✅ Help developers debug if something still fails

ENHANCE BACKEND LOGGING:
  ✅ More detailed request logging
  ✅ Better exception handling

================================================================================
                        KEY LEARNINGS
================================================================================

LESSON 1: FormData Encoding
  ❌ Never manually set Content-Type for FormData
  ✅ Always let the HTTP client (Axios, fetch) set it automatically
  WHY: The client needs to add a boundary marker for proper encoding

LESSON 2: Multipart Format
  Proper multipart request:
    Content-Type: multipart/form-data; boundary=----WebKit123

    ------WebKit123
    Content-Disposition: form-data; name="logo"; filename="logo.png"
    Content-Type: image/png

    [binary image data]
    ------WebKit123
    Content-Disposition: form-data; name="company_name"

    My Company
    ------WebKit123--

LESSON 3: Error Logging
  ❌ Don't catch errors silently
  ✅ Always log errors with context
  WHY: Makes debugging much faster and easier

LESSON 4: Testing
  ❌ Don't just test happy path
  ✅ Test error cases and check logs
  WHY: Catches issues early before production
